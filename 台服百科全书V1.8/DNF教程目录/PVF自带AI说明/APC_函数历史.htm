APC 函数历史

记录该APC的功能和改进的更新历史。
开发者将功能及改善文档的写在最上面的部分，以便留下记录。
2008年7月2日旭。

指定的方法
[ limit passiveobject ]
1 /索引

2007年10月25日更新。
* APC指定功能的深渊派对
1。aic文件中所指定的固定。
-固定深渊派对的成员一开始使用索引
-指定的固定深渊派对不变更
-固定深渊派对可以在实质上没有限制。
指定的方法）
 [party member] // APC序号
  1 20402  // 20402 白虎級抗魔團
  2 5004  // 5004 阿拉德初階格鬥家
  3 605  // 605 骨劍士傑諾
 [/party member]

2。自动登记：特定的不自动登记。
- 0 自己设置。
- 1 - 2 - 3号顺序登记。
-派对成员消失（死的时候），为指数重新补充的成员。
-自动登记的派对可以实质上是没有限制的

3。成员的使用方法
- ai自由派对文件中使用指定自由队员功能
- [set party member]｛成员序号｝对自由派对指定技能对象

4。有关添加功能：AI使用的文件
- [party member alive] : 检查成员是否存活
- [party member hp] ｛参数｝｛数值｝指定成员的HP检查

5。使用：AI文件
[think] [set party member] 2 /设定派对成员为2号
 [think] [party member alive]
  [true]
   [think] [party member hp] `%>` 50
       [true] [return] `a` //返回A值2号队员会使用技能
    [false] [return] -1
   [/think]
  [false]
   [return] -1
 [/think]
[/think]   

2007年10月23日更新。
-ai文件中[check skill on]使用特定技能/时间的检查
- [check skill on]利用技能列表检查“on off skill list rtf”文件。案例
 使用的是)
  [think] [check skill on] 52 // 杀意波动/时间的检查
     [true] //杀意波动开始
    [return] `a`
     [false] //杀意波动消失
    [return] `s`
  [/think]   

2007年10月12日更新。
 - *.ai文件中[return]用名称代替名称 与[return str]行使相同的功能

  [return] `test` //没有问题

2007年10月12日更新。
 - *.aic文件中[key stream]设置名称和按键文件功能
 使用如下)   
 [key stream]
 `combo` `combo.key` //combo是名称，combo.key是文件名

 下面同上
 使用如下)
 [key stream]
 `` `combo.key` //按键名称设置为空，设置消除名称为combo的按键文件名和扩展名的路径传输
 
 另一个使用）
 [key stream]
 `AICharacter/cuwaki/key/combo.key` //不输入按键名称，去除掉，设置按键文件和扩展名的路径

2007年10月10日更新。 
 - key脚本调用不同key文件的功能
 使用方法)[import key] `key 文件名`
   在[import key]的[input]的位置上使用
 注意事项）无限循环
 
 - key文件中的概率开关
 使用方法) [switch], [/switch], [percent] 使用
 使用是) 
  [switch]
   [percent] 50
   [import key] `a.key`
   [percent] 30
   [import key] `s.key`
   [percent] 20
   [exit]
  [/switch]
 注意事项）开关可在语句中直接使用，无法在语句中分开使用, 请使用[import key]
   loop功能也是loop内不能使用。，请使用[import key]。
   一、开关语法里的loop，loop里的开关语法可以使用
   
 - key 强制退出
 使用方法) [exit]
